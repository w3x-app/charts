IMPORTANT: Secret Persistence for Production

⚠️  **JWT Session Errors**: If you're experiencing "JWT_SESSION_ERROR: decryption operation failed", 
this usually means the NEXTAUTH_SECRET was regenerated, invalidating existing sessions.

**Solution**: Set explicit secrets in values.yaml to prevent regeneration:
```yaml
secret:
  enabled: true
  NEXTAUTH_SECRET: "your-fixed-secret-here"  # Generate with: openssl rand -hex 32
  ENCRYPTION_KEY: "your-fixed-key-here"      # Generate with: openssl rand -hex 32
  CRON_SECRET: "your-fixed-cron-secret"      # Generate with: openssl rand -hex 32
```

**To fix existing deployment**:
1. Retrieve current secrets from the cluster
2. Set them explicitly in values.yaml
3. Upgrade the Helm release: `helm upgrade <release-name> . -f values.yaml`

This ensures secrets persist across upgrades and prevents session invalidation.
